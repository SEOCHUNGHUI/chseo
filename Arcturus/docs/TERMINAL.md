## SSH 서버 프로파일 관리

### 서버 추가

좌측 사이드바 상단의 **+ 추가** 버튼을 클릭합니다.

| 필드 | 설명 | 예시 |
|------|------|------|
| 이름 | 식별용 라벨 (선택) | `프로덕션 서버` |
| IP / 호스트 | 서버 주소 (필수) | `192.168.0.75` |
| 포트 | SSH 포트 (기본 22) | `22` |
| 사용자명 | SSH 접속 계정 (필수) | `root` |

> **비밀번호는 저장되지 않습니다.** 접속할 때마다 입력합니다.

### 서버 삭제

서버 목록에서 우측 **✕** 버튼 클릭.

### 프로파일 저장 위치

서버 프로파일은 **브라우저 localStorage**에 저장됩니다.
- 다른 브라우저나 시크릿 모드에서는 목록이 보이지 않습니다
- 브라우저 데이터 초기화 시 삭제됩니다

## 서버 접속 방법

1. 좌측 목록에서 접속할 서버를 클릭
2. 비밀번호 입력 모달에서 SSH 비밀번호 입력
3. **연결** 버튼 클릭 (또는 Enter)
4. 터미널에 셸 프롬프트가 표시되면 접속 완료

### 접속 해제

좌측 하단 **연결 해제** 버튼 클릭, 또는 터미널에서 `exit` 입력.

## 터미널 사용법

표준 UNIX 터미널과 동일하게 사용합니다.

```bash
# 파일 목록
ls -la

# 디렉토리 이동
cd /var/log

# 실시간 로그 확인
tail -f /var/log/app.log

# Docker 명령 (접속된 서버에서)
docker ps
docker logs -f container_name
```

### 터미널 색상 지원

`TERM=xterm-256color`로 설정되므로 컬러 출력을 지원합니다.

## 보안 모델

### 비밀번호 처리 흐름

```
사용자 입력 (비밀번호 모달)
      │
      │  HTTPS (암호화 전송)
      ▼
백엔드 (메모리에서 paramiko 연결에 사용)
      │
      │  SSH 프로토콜
      ▼
원격 서버
      │
비밀번호는 저장되지 않음 — 연결 후 즉시 폐기
```

### 저장되는 정보

| 정보 | 저장 위치 | 암호화 |
|------|-----------|--------|
| 서버 이름 | 브라우저 localStorage | ✗ |
| 호스트/포트 | 브라우저 localStorage | ✗ |
| 사용자명 | 브라우저 localStorage | ✗ |
| **비밀번호** | **저장 안 함** | — |

### 저장되지 않는 정보

- SSH 비밀번호 (세션이 끊기면 사라짐)
- 터미널 입출력 내용
- SSH 세션 기록

### 네트워크 보안

- 패널과 브라우저 간 통신은 Nginx를 통해 HTTPS로 보호 (HTTPS 적용 시)
- SSH 터미널 데이터는 WebSocket(`wss://`)으로 암호화 전송
- 백엔드와 원격 서버 간 통신은 SSH 프로토콜로 암호화

### 위협 모델

| 위협 | 대응 |
|------|------|
| localStorage 탈취 | 비밀번호 미저장 — 호스트/포트/사용자명만 노출 |
| 네트워크 도청 | HTTPS/WSS 암호화 |
| 패널 미인가 접속 | JWT 인증 필수, 로그인 실패 IP 기반 Rate Limit (5분 10회) |
| 세션 탈취 | JWT 만료 시간 설정 (기본 24시간) |

## 제한 사항

- **키 기반 인증 미지원** — 현재 패스워드 인증만 지원
- **SFTP 미지원** — 파일 전송 불가
- **포트 포워딩 미지원**
- **동시 접속** — 하나의 서버에 하나의 세션만 유지 (브라우저 탭당)
