## 기능 개요

| 기능 | 설명 |
|------|------|
| 컨테이너 목록 조회 | 실행 중 + 중지된 모든 컨테이너 표시 |
| 상태 모니터링 | 10초마다 자동 갱신 |
| 컨테이너 제어 | 시작 / 중지 / 재시작 |
| 로그 뷰어 | 실시간 로그 스트리밍 (tail -f 방식) |
| Exec 터미널 | 컨테이너 내부 셸 접속 (자격증명 불필요) |

## 컨테이너 목록

### 컬럼 설명

| 컬럼 | 설명 |
|------|------|
| **Name** | 컨테이너 이름 (Docker Compose의 서비스명 기반) |
| **ID** | 컨테이너 Short ID (12자리 해시) |
| **Image** | 사용 중인 이미지명 및 태그 |
| **Port** | 호스트:컨테이너 포트 매핑. 없으면 `-` |
| **Status** | 현재 상태 (색상으로 구분) |

### 상태 색상

| 색상 | 상태 | 설명 |
|------|------|------|
| 🟢 초록 | `running` | 정상 실행 중 |
| 🔴 빨강 | `exited` / `dead` / `stopped` | 중지됨 |
| 🟡 노랑 | `restarting` / `paused` | 재시작 중 또는 일시 정지 |
| ⚪ 회색 | `created` / `removing` | 생성됨 또는 삭제 중 |

### 자동 갱신

컨테이너 목록은 **10초마다 자동으로 새로고침**됩니다. 즉시 반영이 필요하면 상단 **새로고침** 버튼을 클릭합니다.

## 컨테이너 제어

각 행 우측의 버튼으로 컨테이너를 제어합니다.

| 버튼 | Docker 명령 | 설명 |
|------|------------|------|
| 시작 | `docker start` | 중지된 컨테이너 시작 |
| 중지 | `docker stop` | 실행 중인 컨테이너 정상 종료 (SIGTERM) |
| 재시작 | `docker restart` | 컨테이너 재시작 |

> **주의:** 중지/재시작은 즉시 적용됩니다. 프로덕션 컨테이너 조작 시 서비스 중단이 발생할 수 있습니다.

## 로그 뷰어

컨테이너 클릭 → 하단 **로그** 탭에서 확인.

### 동작 방식

- 최근 **200줄**을 초기 로드합니다
- 이후 새 로그가 발생하면 **실시간으로 추가** (WebSocket 스트리밍)
- 화면 하단에 있으면 **자동 스크롤** (tail -f 동작)
- 로그를 위로 스크롤하면 자동 스크롤이 **일시 중지**
- 하단으로 다시 스크롤하면 자동 스크롤 **재개**

### 팁

```
로그가 너무 빠르게 내려가면 → 스크롤을 위로 올리면 일시정지
특정 컨테이너 로그만 보고 싶으면 → 해당 컨테이너 행 클릭
```

## Exec 터미널

컨테이너 클릭 → 하단 **터미널 exec** 탭.

### 동작 방식

별도의 자격증명 없이 Docker API를 통해 컨테이너 내부 셸에 접속합니다.

```
브라우저 → WebSocket → 백엔드 → docker exec → 컨테이너 셸
```

### 셸 자동 감지

컨테이너 내 사용 가능한 셸을 순서대로 탐지합니다:

1. `/bin/bash` (Ubuntu, Debian 등)
2. `/bin/ash` (Alpine)
3. `/bin/sh` (최소 이미지)

### 사용 예시

```bash
# 환경변수 확인
env | grep DB

# 프로세스 확인
ps aux

# 네트워크 확인
netstat -tlnp   # 또는 ss -tlnp

# 파일 확인
cat /app/config.yaml

# Python/Node 실행
python3 manage.py shell
node -e "console.log(process.env)"
```

### 연결 안 될 때

- 컨테이너 상태가 `running`이어야 합니다
- 컨테이너가 최소 이미지(`scratch`, `distroless`)이면 셸이 없어 접속 불가
- 로그 탭에서 에러 메시지 확인

## 보안 모델

### Docker 소켓 접근

Arcturus 백엔드는 `/var/run/docker.sock`을 마운트하여 Docker를 제어합니다.

```
백엔드 컨테이너 → /var/run/docker.sock → Docker 데몬 → 모든 컨테이너 제어 가능
```

이는 **호스트 루트 수준의 권한**에 해당합니다. 따라서:

- Arcturus 패널 접근을 **신뢰할 수 있는 사용자로만 제한**
- 방화벽 또는 VPN으로 패널 포트 접근 제한 권장
- 패널 계정 공유 금지

### Exec 터미널 보안

- Docker exec는 컨테이너 내부에서 **루트로 실행**될 수 있습니다
- exec로 진입 후 컨테이너 탈출(escape)은 컨테이너 보안 설정에 따라 다름
- `--privileged` 플래그로 실행된 컨테이너는 특히 주의 필요

### 감사 로그

현재 버전에서는 컨테이너 제어 액션에 대한 감사 로그가 없습니다. 중요한 환경에서는 별도 로깅 시스템 연동을 권장합니다.

## 제한 사항

- **볼륨/네트워크 관리 불가** — 컨테이너 제어만 지원 (start/stop/restart)
- **이미지 관리 불가** — pull, build, remove 불가
- **Docker Compose 프로젝트 인식 불가** — 개별 컨테이너 단위로만 표시
- **로그 검색 불가** — 텍스트 필터링 미지원
- **로그 다운로드 불가**
